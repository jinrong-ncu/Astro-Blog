资料丢失这件事，通常只需要发生一次，就会让人终身学会备份。

课程论文、实验报告、项目代码、证件照片、简历、作品集，这些文件平时看起来普通，真丢了会非常麻烦。学生阶段最容易犯的错，是把所有资料都放在一个地方：桌面、学校网盘、某个 U 盘，或者一个快满了的手机相册。

## 先区分文件重要性

不是所有文件都需要同等备份。

**必须备份**：论文、简历、证件、项目代码、作品集、重要截图、长期笔记。

**建议备份**：课程课件、作业、实验数据、读书资料、社团文件。

**可不备份**：临时下载、重复安装包、随手保存的无用图片。

先分级，备份才不会变成“把垃圾复制三份”。

## 用 3-2-1 思路

经典备份思路是 3-2-1：

- 至少 3 份副本
- 放在 2 种不同介质
- 至少 1 份不在同一台设备上

学生不一定要做得特别专业，但可以简化成：

- 电脑本地一份
- 云盘同步一份
- 移动硬盘或另一个云盘再放一份重要资料

这样电脑坏了、云盘误删了、账号出问题了，都不至于全没。

## 学校账号不要当唯一备份

很多学校提供 Microsoft 365、OneDrive 或邮箱网盘。它们很好用，但不应该是唯一备份位置。

原因很简单：毕业后账号可能失效，容量策略可能变化，学校管理员也可能调整服务。课程资料可以放学校账号，长期私人资料最好同步到个人账号或本地硬盘。

## 代码项目用 Git

如果你学编程，项目代码不要只放压缩包。

用 Git 管理版本，再推到 GitHub 或其他代码托管平台。这样你不仅能备份代码，还能保留修改历史。哪怕某次改坏了，也能回到之前版本。

每个项目最好写一个 README，说明项目是什么、怎么运行、用了什么技术。以后整理作品集时会省很多时间。

## 一个可执行的备份方案

可以这样做：

1. 桌面只放临时文件
2. 所有学习资料放到 `Documents/Study`
3. `Study` 同步到 OneDrive、iCloud 或其他个人云盘
4. 每个月把重要文件复制到移动硬盘
5. 代码项目全部用 Git 管理
6. 毕业前导出学校账号里的所有资料

这个方案不复杂，但已经能避免大多数灾难。

## 我的建议

备份的关键不是工具，而是习惯。

你不需要买很贵的 NAS，也不需要研究复杂同步规则。先做到两件事：重要文件不放桌面，学校账号不当唯一备份。只要做到这两点，资料安全就已经提高很多。
