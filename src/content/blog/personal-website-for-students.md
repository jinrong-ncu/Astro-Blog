很多学生等到找实习时，才开始整理作品集。结果项目截图找不到，代码仓库没 README，课程项目只有压缩包，简历上写了很多技能，但没有一个地方能证明。

个人网站解决的不是“看起来高级”的问题，而是把你的学习过程长期保存下来。它可以很简单：几个项目页、几篇笔记、一份关于自己页面，就够了。

## 个人网站可以放什么

最适合学生放的内容有四类。

**项目记录**：课程项目、个人小工具、开源练习、比赛作品。每个项目说明背景、技术栈、你负责的部分、遇到的问题和最终结果。

**学习笔记**：不要把所有课堂笔记都发出来，挑能帮别人解决问题的内容。比如某个环境配置、某个框架踩坑、某个概念解释。

**作品集**：设计图、前端页面、数据分析报告、论文摘要、视频作品，都可以整理成独立页面。

**个人介绍**：你是谁，学什么，关注什么，正在做什么，怎么联系你。

## 为什么不用只发社交平台

社交平台适合传播，但不适合长期归档。

平台规则会变，内容入口会变，搜索能力也不一定稳定。个人网站的好处是结构由你控制：URL、标题、标签、RSS、站点地图、搜索引擎收录，都可以按自己的节奏维护。

更重要的是，个人网站能体现连续性。别人看到的不只是某一条动态，而是你过去一年、两年持续做过什么。

## 技术上怎么开始

最简单的路线是：

1. 用 Astro、Next.js 或 Hugo 搭一个静态网站
2. 内容用 Markdown 写
3. 部署到 Vercel、Cloudflare Pages 或 GitHub Pages
4. 配置自定义域名
5. 添加站点地图、RSS 和基础 SEO

如果你不会写代码，也可以先用 Notion、语雀、GitHub Pages 模板或其他无代码工具开始。重点不是技术栈，而是先开始积累内容。

## 第一批内容写什么

不要一开始就写“关于我未来十年的宏大计划”。更好的起点是这三篇：

- 我是怎么搭建这个网站的
- 我最近完成的一个课程项目
- 我踩过的一个技术坑

这三类内容都比较真实，也容易写出细节。搜索引擎和读者都更喜欢有具体问题、有解决过程、有个人判断的内容。

## 我的建议

个人网站是一个慢变量。它不会让你一夜之间变得厉害，但会把你每一次学习和实践留下来。

等你需要找实习、申请项目、介绍自己时，一个持续维护的网站会比临时拼出来的作品集自然得多。你不是在展示“我会什么”，而是在展示“我一直在做什么”。
